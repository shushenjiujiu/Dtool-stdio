# 代码交付证据标准 (Delivery Evidence Standard)

> 制定人：助手二号 | 版本 v0.1 | 2025-06-26
>
> 依据原则：**独立验证** — 产出必须由产出者以外的一方验证通过，否则视为未交付。

---

## 1. 规则

**代码交付必须附带可复现的验证证据。** "build 通过"不是证据——它是前提。

## 2. 证据等级

### L1：必须附带（缺一则交付退回）

| 证据项 | 具体要求 | 不合格示例 |
|--------|----------|-----------|
| 类型检查 | `tsc --noEmit` 完整输出，末尾必须显示 `(no output)` 或零错误 | 只说"type-check 通过"无输出 |
| 测试结果 | `npx vitest run` 完整输出，包含 test files / tests 计数行 | 只说"98 tests passed"无完整列表 |
| 构建产物 | `npm run build` 退出的 exit code 为 0，无 error 级别日志 | 只说"build 通过" |

### L2：强烈建议（缺一则交付标记为"未验证"）

| 证据项 | 具体要求 |
|--------|----------|
| 端到端 Smoke | 一条可复现的命令或浏览器操作步骤，验证核心流程可用 |
| 变更影响范围 | 列出受影响的文件/模块/接口，防止集成断裂 |

### L3：按需附加

| 证据项 | 适用场景 |
|--------|----------|
| 性能基准 | 涉及循环/大数据量处理的变更 |
| 安全审计摘要 | 涉及输入验证、注入防护的变更 |
| 迁移步骤 | 涉及数据格式/API 变更 |

---

## 3. 证据格式

### 3.1 交付消息模板

```
## 交付：{功能名称}

### 变更摘要
- 新增/修改文件列表
- 影响的接口/模块

### L1 证据

**类型检查**
$ cd packages/engine && npx tsc --noEmit
(退出码 0，无输出)

**测试**
$ npx vitest run
✓ Test Files  6 passed (6)
     Tests  98 passed (98)

**构建**
$ npm run build
(退出码 0)

### L2 证据

**端到端 Smoke**
{一条可复现的命令或操作步骤，含预期输出}
```

### 3.2 端到端 Smoke 模板

```bash
# 1. 启动服务
cd packages/server && npm start &

# 2. 执行核心流程
curl -s http://localhost:3000/api/health | jq .

# 3. 验证预期输出
# {"status":"ok"}  ← 预期结果
```

或（前端变更）：

```
# 1. 打开 http://localhost:5173
# 2. 创建管道：拖入 Base64 编码 → URL 编码
# 3. 输入文本 "test" → 点击运行
# 4. 预期：输出区域显示编码结果
```

---

## 4. 判定规则

| 交付证据状态 | 判定 | 动作 |
|-------------|------|------|
| L1 三项齐全 + L2 Smoke | ✅ 验收通过 | 任务关闭 |
| L1 齐全，L2 缺失 | ⚠️ 验收暂缓 | 标记"未验证"，24h 内补 Smoke |
| L1 缺任意一项 | ❌ 退回 | 附注"证据不足"，要求重交 |
| 三次 L1 不全 | 🚫 信用降级 | 该 Agent 后续产出默认不采信 |
